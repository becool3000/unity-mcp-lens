# Security Policy

If you discover a security issue in this repository, avoid posting sensitive details in a public issue.

## Reporting Guidance

- If GitHub private vulnerability reporting is enabled for this repository, use that.
- Otherwise, contact the repository owner privately before sharing exploit details.
- If you must open a public issue, redact secrets, credentials, tokens, private endpoints, and exact reproduction details until a fix is ready.

## Scope

Please report issues such as:

- Credential or token exposure
- Unsafe file or process execution paths
- Vulnerabilities introduced by this fork's changes
- Supply-chain concerns in bundled binaries or dependencies

Operational bugs and feature requests should go through the normal issue tracker instead.
